WorkRest 更新需求文档
更新时间：2026-03-04

版本 2.0.1：

1）窗口控制
- 在右上角增加一个“全局窗口”按钮（当前只有最小化和关闭）。
- 按钮风格与现有最小化、关闭按钮保持一致。

2）统计页「时间段分布」规则调整
- 时间段边界固定为：
	- 早上：09:00-12:00（目标 3 小时）
	- 下午：12:00-18:00（目标 4 小时）
	- 晚上：18:00-23:00（目标 3 小时）
- UI 柱状段位置固定为：
	- 黄色（早上）在最前段（0h-3h）
	- 蓝色（下午）放在 3h-7h 区间，从 3h 起加载
	- 紫色（晚上）从 7h 起（对应 18:00 后）
- 工作时长归属规则：按「开始时刻」归属，不按结束时刻。
	- 示例：11:50 开始，12:35 结束，整段工作时长都计入早上。
- 超时规则（重点）：
	- 早上超过 3h、下午超过 4h、晚上超过 3h 的超出部分，不继续把柱状条拉长；
	- 超出部分在对应时段内显示为红色（例如早上超出就早上段内变红）。
- 下方信息框（“XX小时XX分钟 XX%”）规则：
	- 显示该时段实际累计时间；
	- 当该时段达到或超过目标时，百分比显示 100%；
	- 原有文本内容不变，仅增加“熊熊燃烧的烈火”视觉特效，用于表现非常努力。

3）工作阶段提示音
- 开始工作时播放一次“叮”（音量不要太大）。
- 在「工作总时长」剩余 2/3 时播放一次“叮叮”。
- 在「工作总时长」剩余 1/3 时播放一次“叮叮叮”。
- 以上提示音均受静音开关控制。

4）休息结束自动停音乐
- 当前问题：休息结束后点击开始工作，音乐又会开始。
- 新逻辑：休息时间一结束，立即自动停止音乐（不管用户有没有点按钮）。
- 若休息结束自动进入下一轮工作：
	- 先停止休息音乐；
	- 然后按规则触发“开始工作”的“叮”提示音。

5）默认音乐与目录逻辑
- 项目内提供默认音乐：NIKON【I AM】（默认循环播放此曲目）。
- 继续保留“读取用户目录”的现有逻辑不变：
	- 用户未选择目录时：循环播放默认音乐；
	- 用户选择目录后：随机播放目录下全部歌曲。
- 提示音“叮/叮叮/叮叮叮”音频文件也放在项目目录下。（两声和三声就循环播放就行）

6）中文语音优化
- 将中文语音替换为更自然、更好听的音色（风格接近“小爱同学”）。

7）README 图片显示修复
- 修复 GitHub 页面中 README 图片不显示问题（当前只显示“WorkRest Logo”文字）。
- 提交前需在 GitHub 渲染结果中确认图片可正常显示。

8）修复 mini / max 冲突
- 解决 mini 版本无法打开、max 正常的冲突问题。
- 目标：mini 与 max 均可独立正常启动。

9）Release 发布规范（后续长期执行）
- 每次推送 GitHub 时，同步发布 Releases，且包含：
	- mini 打包产物
	- max 打包产物
	- 对应源码包
    - 打包为完整的deb包，应该是几十mb大小的

10）更新后完整测试与应用可见性
- 更新完成后进行完整测试（至少覆盖核心流程：工作、休息、统计、音乐、语音、mini/max 启动）。
- 打包为应用程序后，需能在系统应用列表中被找到并启动。

- 后续每次都直接读取本文件作为更新输入。
